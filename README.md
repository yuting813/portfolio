## 我專注的工程面向

* **UI 架構設計**：元件保持純粹，狀態與副作用集中管理
* **狀態可預測性**：登入、權限、非同步資料流程不閃爍、不猜測
* **UX 邊界情境**：明確處理 loading / empty / error 狀態
* **效能導向**：以 CLS / LCP 為指標的實際優化
* **工程流程**：Git Flow、語義化 Commit、可回溯的 PR 說明

## 代表專案

###  MERN 課程平台（全端）

> 以「權限驅動 UI」為核心的全端專案，狀態集中管理而非分散判斷。

**Live**：[https://course.tinahu.dev](https://course.tinahu.dev)
**Code**：[https://github.com/yuting813/mern-project](https://github.com/yuting813/mern-project)

**技術棧**
MongoDB · Express · React · Node.js · TypeScript · JWT · Joi

**專案重點**

* 登入與角色權限集中管理，避免元件各自判斷
* 角色切換與初始化過程中，UI 行為穩定不閃爍
* Joi Schema 模組化表單驗證，前後端規則一致
* API 錯誤集中處理，避免畫面出現零散錯誤

---

### Netflix Clone（前端）

> 著重狀態可預測性與 UX 邊界條件的前端系統設計。

**Live**：[https://stream.tinahu.dev](https://stream.tinahu.dev)
**Code**：[https://github.com/yuting813/netflix-clone-nextjs](https://github.com/yuting813/netflix-clone-nextjs)

**技術棧**
Next.js · TypeScript · Tailwind CSS · Firebase · TMDB API · Stripe

**專案重點**

* 登入與訂閱狀態改變時，UI 行為保持一致
* 明確區分 loading / empty / error 狀態
* 圖片與資料載入順序設計，實際降低 CLS
* modal 與背景內容狀態分離，避免互相干擾
